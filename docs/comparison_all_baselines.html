Learn2Smile - Baselines

Here we are comparing the results from our model vs. baselines for 30 randomly chosen ground truth videos. The baselines are sorted by their effectiveness using the "L2 Error w/ Shift" measure detailed in the paper.

On the left side:
The 1st column shows the ground truth of the user's facial expressions.
The 2nd column shows the ground truth response of the agent.
The 3rd column shows the predicted response from our model, which has two predictors for low-frequency and high-frequency components respectively, and uses both user's and agent's history as input signal.

On the right side:
The 4th column shows the predicted response from a baseline that randomly chooses an agent sequence and starting position from the training set, without using any of the user's input.
The 5th column shows the predicted response from a baseline that uses a 30-frame temporal average of the user's input as response.
The 6th column shows the predicted response from a baseline that copies the user's facial expression exactly, and delays it by 90 frames.
The 7th column shows the predicted response from a baseline that finds the closest user segment in the training set and returns its corresponding agent segment.

Note that the 30 videos shown are randomly selected from the hold-out test set.

Discussion:
Since the first three baselines are either not using user's input or not using it in a task-relevant way, we focus our discussion on the last baseline (nearest neighbor). On a high level, we can notice that the predicted face jumps between different short sequences, which is due to the fact that the closest user segment match at different timesteps can be found in different videos of the training set, hence the corresponding response from agent is not from the same person and will not be consistent.
